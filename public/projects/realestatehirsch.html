Core experience

Built to keep prospects moving toward a confirmed tour.

Story-driven landing

Hero imagery, testimonials, and quick stats establish trust while keeping the primary tour CTA in view across breakpoints.

Interactive scheduling

Flatpickr calendar with client-side restrictions, real-time availability checks via API, and server-side validation prevent double bookings. All bookings are saved to the database and duplicate prevention works as expected.

Easy updates

Reusable content modules let agents refresh availability and promotions without editing code.

Under the hood

Lightweight backend services keep the team responsive.

RESTful API

Express.js endpoints with parameterized queries prevent SQL injection. Connection pooling optimizes PostgreSQL performance on Neon's serverless platform.

Database optimization

Strategic indexes on date, property, and email columns enable fast conflict detection and availability checks. Automatic cleanup maintains performance.

Email automation

Current limitation: Render's free tier blocks outbound SMTP ports (25, 587, 465), so confirmation emails cannot be sent. However, the booking system fully functions—all bookings are saved to the database, duplicate bookings are prevented, and cancellation links work via HTTP endpoints (not email-dependent).

To enable emails: Upgrade Render to paid tier (enables SMTP) or migrate to an API-based email service like SendGrid or Mailgun that uses HTTPS instead of SMTP.
