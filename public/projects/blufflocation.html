Realtime Party Game

BluffLocation

A mobile social-deduction game where one player bluffs and everyone else hunts for the spy.

Everyone gets the same location prompt except the spy. Players talk it out, vote, and the spy gets a last-chance guess to steal the round. The whole flow stays in sync through a Socket.IO websocket so every phone sees the same room state, timer, and vote outcomes.

What the app does

Designed for quick, in-person rounds that don’t require setup or a moderator.

Room-based play

Hosts create a room code, friends join instantly, and the lobby stays updated as players arrive or leave.

Guided round flow

The host starts the round, a server timer runs, and the UI moves players from discussion → voting → reveal without confusion.

Votes + last-chance guess

Votes are recorded server-side and revealed as soon as everyone has submitted. If the spy survives, they get one final guess.

How the websocket keeps it fair

The server is the source of truth so no device can “get ahead” or disagree about outcomes.

Room lifecycle

Clients emit events to create and join rooms, while the server broadcasts room updates (like players joining/leaving) to everyone watching that room.

Authoritative game state

The server assigns roles, starts the round, and pushes state updates (timer, locations, votes, results) so every client renders the same truth at the same time.

Reconnect + resync

If a player drops, the client reconnects, re-joins the room, and requests a full state sync snapshot so they can resume without breaking the round.

Next steps

Easy wins to turn the prototype into a “game night staple.”

  • More location packs + custom packs for groups.
  • Host controls (kick/lock room) and lightweight moderation.
  • Better reconnection UX (toast + quick resync indicator).
